# 通信协议

## 概述

Univona 的通信分为三层：

1. **REST/HTTPS**：用于认证、频道管理、媒体上传等请求响应式操作
2. **WebSocket**：用于实时消息投递与在线状态
3. **P2P/WebRTC**：用于端到端直连的实时通信

所有实时消息均采用 **Protobuf Envelope** 二进制协议编码。

## REST/HTTPS

- 统一前缀：`/api/v1/...`
- 响应格式：`{ success, data, error }`
- 认证：Bearer JWT 或 X-Admin-Key

## WebSocket 实时通信

- 连接地址：`/ws?token=<user_jwt>`
- 仅支持 Binary 帧（Protobuf `Envelope`）
- 服务器每 30 秒发送 Ping，90 秒无响应断开

## 消息类型

Protobuf `MessageType` 主要分为：

- 内容消息：`TEXT` / `IMAGE` / `FILE` / `VOICE` / `VIDEO`
- 密钥交换：`KEY_EXCHANGE` / `PREKEY_BUNDLE`
- 临时消息：`READ_RECEIPT` / `TYPING_INDICATOR`
- 系统消息：`SYSTEM` / `GROUP_MEMBER_CHANGE`
- P2P 信令：`P2P_OFFER` / `P2P_ANSWER` / `P2P_ICE_CANDIDATE`

## 加密策略

频道级别支持三种策略：

- `plaintext`：明文存储，便于审计与搜索
- `e2e_required`：强制端到端加密
- `admin_encrypted`：管理员启用加密

当策略为加密时，服务端仅保存密文并禁止明文索引。
