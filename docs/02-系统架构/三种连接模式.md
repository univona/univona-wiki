# 三种连接模式

## 概述

Univona 支持三种部署/连接模式，覆盖从完全去中心化到官方中枢联邦的不同需求。客户端可以同时连接多种模式，按场景选择通信路径。

## Mode A - 完全 P2P

**特点**：客户端之间直接通信，不依赖任何服务器。

- 传输：WebRTC / libp2p
- 身份：本地 Ed25519 密钥对
- 适用：局域网、高隐私需求、临时沟通
- 限制：NAT 穿透依赖 STUN/TURN，离线消息能力有限

## Mode B - 社区 Daemon

**特点**：单社区自托管服务器，类似 Discord 服务器模型。

- 传输：客户端 ↔ Daemon WebSocket + REST
- 服务端职责：频道管理、消息路由、媒体存储、离线中继
- 适用：社区自治、组织自托管
- 优势：部署简单、数据主权清晰

## Mode C - Admin Server 联邦

**特点**：由官方或组织运营的中枢服务器，管理多个 Daemon 并提供目录服务。

- 传输：客户端 ↔ Admin Server（目录/KYC） + 客户端 ↔ Daemon（实时消息）
- 额外能力：社区目录、KYC、跨社区发现、内容监控
- 适用：需要统一入口与合规管理的大型部署

## 模式对比

| 维度 | Mode A (P2P) | Mode B (Daemon) | Mode C (Admin Server) |
|------|-------------|-----------------|------------------------|
| 部署复杂度 | 低 | 中 | 高 |
| 数据主权 | 最高 | 高 | 中 |
| 功能完整度 | 中 | 高 | 最高 |
| 合规能力 | 低 | 中 | 高 |
| 目录/发现 | 无 | 社区内 | 全局 |
