# 认证流程

## 1. Admin Panel JWT 认证

管理面板使用传统的用户名/密码登录，服务端签发 JWT。

### 登录流程

```text
客户端                          服务端
  |                               |
  |  POST /api/v1/admin/login     |
  |  { username, password }       |
  |------------------------------>|
  |                               | 验证密码 (bcrypt)
  |                               | 签发 Admin JWT (8h 有效期)
  |  { success, token,            |
  |    expires_at, username,      |
  |    role }                     |
  |<------------------------------|
  |                               |
  |  GET /api/v1/admin/...        |
  |  Authorization: Bearer <jwt>  |
  |------------------------------>|
  |                               | 验证 JWT
  |  200 OK                       |
  |<------------------------------|
```

### 登录请求

```bash
curl -X POST http://localhost:3000/api/v1/admin/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "your_password"
  }'
```

### 登录响应

```json
{
  "success": true,
  "username": "admin",
  "role": "super_admin",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires_at": 1709128800
}
```

### Admin JWT Claims

```json
{
  "sub": "<admin_uuid>",
  "username": "admin",
  "role": "super_admin",
  "token_type": "admin_access",
  "iat": 1709100000,
  "exp": 1709128800
}
```

### 首次启动自动创建

当 `admins` 表为空时，首次登录会自动创建默认管理员账户：

- 用户名：`admin`
- 密码：`univona`
- 角色：`super_admin`

> **安全提示**：部署后请务必立即修改默认密码。

### 向后兼容：X-Admin-Key

除 JWT 外，仍支持 `X-Admin-Key` 头进行程序化认证。中间件按以下顺序检查：

1. `Authorization: Bearer <jwt>` → 验证 JWT → 通过
2. `X-Admin-Key: <key>` → 匹配环境变量 `ADMIN_KEY` → 通过
3. 两者都无 → `401 Unauthorized`

```bash
# 使用 X-Admin-Key
curl http://localhost:3000/api/v1/admin/daemons \
  -H "X-Admin-Key: your_admin_key"
```

## 2. 用户 Challenge-Response 认证

Daemon 用户使用基于 Ed25519 签名的挑战-响应机制认证。

### 注册流程

```text
客户端                          服务端
  |                               |
  | 本地生成 Ed25519 密钥对        |
  |                               |
  |  POST /api/v1/auth/register   |
  |  { username, identity_key,    |
  |    display_name, device }     |
  |------------------------------>|
  |                               | 验证用户名和密钥
  |                               | 存储 identity_key
  |                               | 创建 member + device
  |                               | 签发 access + refresh token
  |  { access_token,              |
  |    refresh_token,             |
  |    expires_at, member_id }    |
  |<------------------------------|
```

### 注册请求

```bash
curl -X POST http://localhost:3000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "alice",
    "identity_key": "<32字节Ed25519公钥的hex编码>",
    "display_name": "Alice",
    "device": {
      "device_id": "<uuid>",
      "device_name": "iPhone 15",
      "platform": "ios",
      "device_identity_key": "<hex>",
      "signed_prekey": "<hex>",
      "signed_prekey_signature": "<hex>",
      "signed_prekey_id": 1
    }
  }'
```

### 注册响应

```json
{
  "success": true,
  "data": {
    "access_token": "eyJ...",
    "refresh_token": "eyJ...",
    "expires_at": 1709103600,
    "member_id": "019f1234-5678-7abc-def0-123456789abc"
  }
}
```

### 登录流程（Challenge-Response）

```text
客户端                          服务端
  |                               |
  |  POST /api/v1/auth/challenge  |
  |------------------------------>|
  |                               | 生成 32 字节随机 nonce
  |                               | 存入 auth_challenges (5分钟过期)
  |  { challenge_id, nonce,       |
  |    expires_at }               |
  |<------------------------------|
  |                               |
  | 用 Ed25519 私钥签名 nonce     |
  |                               |
  |  POST /api/v1/auth/verify     |
  |  { challenge_id, identity_key,|
  |    signature, device_id }     |
  |------------------------------>|
  |                               | 查找 challenge
  |                               | 验证 Ed25519 签名
  |                               | 标记 challenge 为已用（防重放）
  |                               | 签发 JWT
  |  { access_token,              |
  |    refresh_token,             |
  |    expires_at, member_id }    |
  |<------------------------------|
```

#### 步骤一：请求挑战

```bash
curl -X POST http://localhost:3000/api/v1/auth/challenge
```

响应：

```json
{
  "success": true,
  "data": {
    "challenge_id": "uuid-of-challenge",
    "nonce": "<32字节随机数的hex编码>",
    "expires_at": 1709100300
  }
}
```

#### 步骤二：验证签名

```bash
curl -X POST http://localhost:3000/api/v1/auth/verify \
  -H "Content-Type: application/json" \
  -d '{
    "challenge_id": "uuid-of-challenge",
    "identity_key": "<Ed25519公钥hex>",
    "signature": "<签名hex>",
    "device_id": "device_uuid"
  }'
```

响应：

```json
{
  "success": true,
  "data": {
    "access_token": "eyJ...",
    "refresh_token": "eyJ...",
    "expires_at": 1709103600,
    "member_id": "member-uuid"
  }
}
```

### 用户 JWT Claims

**Access Token** (1 小时有效):

```json
{
  "sub": "<member_uuid>",
  "device_id": "device_abc123",
  "token_type": "access",
  "iat": 1709100000,
  "exp": 1709103600
}
```

**Refresh Token** (30 天有效):

```json
{
  "sub": "<member_uuid>",
  "device_id": "device_abc123",
  "token_type": "refresh",
  "iat": 1709100000,
  "exp": 1711692000
}
```

### 令牌刷新

当 Access Token 过期时，使用 Refresh Token 获取新的令牌对。

```bash
curl -X POST http://localhost:3000/api/v1/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{
    "refresh_token": "<refresh_token>"
  }'
```

响应：

```json
{
  "success": true,
  "data": {
    "access_token": "<new_access_token>",
    "refresh_token": "<new_refresh_token>",
    "expires_at": 1709107200,
    "member_id": "member-uuid"
  }
}
```

> 注意：刷新操作会同时返回新的 refresh_token，旧的 refresh_token 即失效。

## 安全设计要点

- **密钥隔离**：Admin JWT 和用户 JWT 使用**不同的 secret**（`JWT_SECRET` vs `UNIVONA_JWT_SECRET`），防止跨系统伪造。
- **防重放攻击**：认证挑战有 5 分钟过期时间和单次使用限制。
- **端到端认证**：Ed25519 签名确保只有持有私钥的客户端才能认证，服务端永远不接触用户的私钥。
- **密码安全**：管理员密码使用 bcrypt 哈希存储，不可逆。
- **登录限流**：管理员登录端点使用独立的速率限制中间件，防止暴力破解。
- **CORS 保护**：可通过配置限制允许的跨域来源。
