# 开发路线图

## 概述

Univona 采用阶段式迭代路线图，覆盖身份、消息系统、社区服务器与官方中枢等能力。

## 阶段概览

| 阶段 | 目标 | 状态 |
|------|------|------|
| Phase 0 | 项目搭建、协议与 CI | ✅ 完成 |
| Phase 1 | 身份与加密核心 | ✅ 完成 |
| Phase 2 | P2P 通信 | ✅ 完成 |
| Phase 3 | 社区服务器 | ✅ 完成 |
| Phase 4 | 目录服务器与 KYC | ✅ 完成 |
| Phase 5 | 全平台发布与打磨 | 计划中 |

## 重点里程碑

- P2P 信令服务完成（WebRTC 信令中继）
- Admin Server 集成 Panel + Daemon + 目录/KYC
- 客户端 Flutter + Rust 核心打通
