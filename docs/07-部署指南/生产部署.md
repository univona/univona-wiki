# 生产部署

本指南提供将 Univona Admin Server 部署到生产环境的完整步骤和最佳实践。

## 系统要求

| 组件 | 最低要求 | 推荐配置 |
|------|----------|----------|
| CPU | 2 核 | 4 核+ |
| 内存 | 2 GB | 4 GB+ |
| 磁盘 | 20 GB SSD | 100 GB+ SSD |
| 操作系统 | Ubuntu 22.04 / Debian 12 | Ubuntu 24.04 LTS |
| PostgreSQL | 15 | 16+ |
| Rust | 1.92+ | 最新稳定版 |

## 安全加固清单

部署到生产环境前，**必须**完成以下安全检查：

- [ ] 修改默认管理员密码（默认 `admin` / `univona`）
- [ ] 设置强随机 `JWT_SECRET`（至少 32 字节随机字符串）
- [ ] 设置强随机 `UNIVONA_JWT_SECRET`（至少 32 字节随机字符串）
- [ ] 限制 `CORS` 源列表为实际域名（不要在生产环境使用 `["*"]`）
- [ ] 配置 TLS（参考 [nginx + TLS 配置](./Nginx-TLS.md)）
- [ ] 限制数据库访问为本地连接或专用网络
- [ ] 配置防火墙规则，仅开放必需端口（80/443）
- [ ] 定期更新系统和依赖
- [ ] 启用审计日志并定期审查

生成强随机密钥：

```bash
# 使用 openssl 生成 32 字节随机密钥
openssl rand -base64 32

# 或使用 /dev/urandom
head -c 32 /dev/urandom | base64
```

## PostgreSQL 生产配置

### 创建专用数据库用户

```bash
sudo -u postgres psql
```

```sql
-- 创建专用用户
CREATE USER univona WITH PASSWORD 'your-strong-password';

-- 创建数据库
CREATE DATABASE univona OWNER univona;

-- 限制权限
REVOKE ALL ON DATABASE univona FROM PUBLIC;
GRANT CONNECT ON DATABASE univona TO univona;
```

### PostgreSQL 性能调优

编辑 `/etc/postgresql/16/main/postgresql.conf`：

```ini
# 连接配置
max_connections = 100
listen_addresses = 'localhost'

# 内存配置（假设 4 GB 内存）
shared_buffers = 1GB
effective_cache_size = 3GB
work_mem = 16MB
maintenance_work_mem = 256MB

# WAL 配置
wal_buffers = 16MB
min_wal_size = 1GB
max_wal_size = 4GB
checkpoint_completion_target = 0.9

# 查询优化
random_page_cost = 1.1          # SSD 存储
effective_io_concurrency = 200  # SSD 存储
default_statistics_target = 100

# 日志
log_min_duration_statement = 1000  # 记录超过 1 秒的慢查询
log_checkpoints = on
log_connections = on
log_disconnections = on
```

### 配置访问控制

编辑 `/etc/postgresql/16/main/pg_hba.conf`：

```
# 仅允许本地连接
local   univona     univona                         scram-sha-256
host    univona     univona     127.0.0.1/32        scram-sha-256
host    univona     univona     ::1/128             scram-sha-256
```

重启 PostgreSQL：

```bash
sudo systemctl restart postgresql
```

## Univona.toml 生产配置

```toml
host = "127.0.0.1"   # 仅监听本地（通过 nginx 反向代理）
port = 3001

[database]
url = "postgres://univona:your-strong-password@localhost/univona"
max_connections = 20

[security]
cors_origins = ["https://your-domain.com", "https://admin.your-domain.com"]

[log]
level = "info"
json = true   # 生产环境推荐 JSON 格式日志

[push]
[push.fcm]
project_id = "your-gcp-project-id"
service_account_path = "/etc/univona/fcm-service-account.json"

[push.apns]
key_path = "/etc/univona/apns-key.p8"
key_id = "YOUR_KEY_ID"
team_id = "YOUR_TEAM_ID"
bundle_id = "com.your.app"
sandbox = false
```

## systemd 服务文件

创建 `/etc/systemd/system/univona-admin-server.service`：

```ini
[Unit]
Description=Univona Admin Server
After=network.target postgresql.service
Requires=postgresql.service

[Service]
Type=simple
User=univona
Group=univona
WorkingDirectory=/opt/univona
ExecStart=/opt/univona/univona-admin-server
Restart=on-failure
RestartSec=5
StartLimitBurst=3
StartLimitIntervalSec=60

# 环境变量
Environment=RUST_LOG=info
Environment=JWT_SECRET=your-admin-jwt-secret
Environment=UNIVONA_JWT_SECRET=your-user-jwt-secret
Environment=MEDIA_STORAGE_DIR=/var/lib/univona/media

# 安全加固
NoNewPrivileges=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/var/lib/univona /var/log/univona
PrivateTmp=true
ProtectKernelTunables=true
ProtectKernelModules=true
ProtectControlGroups=true

# 资源限制
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
```

### 部署步骤

```bash
# 创建系统用户
sudo useradd -r -s /sbin/nologin -d /opt/univona univona

# 创建目录
sudo mkdir -p /opt/univona /var/lib/univona/media /var/log/univona
sudo chown -R univona:univona /opt/univona /var/lib/univona /var/log/univona

# 复制二进制文件和配置
sudo cp target/release/univona-admin-server /opt/univona/
sudo cp Univona.toml /opt/univona/

# 启用并启动服务
sudo systemctl daemon-reload
sudo systemctl enable univona-admin-server
sudo systemctl start univona-admin-server

# 查看状态和日志
sudo systemctl status univona-admin-server
sudo journalctl -u univona-admin-server -f
```

## 日志管理

### journald 配置

编辑 `/etc/systemd/journald.conf`：

```ini
[Journal]
SystemMaxUse=2G
SystemMaxFileSize=100M
MaxRetentionSec=30day
```

### 日志轮转（如输出到文件）

创建 `/etc/logrotate.d/univona`：

```
/var/log/univona/*.log {
    daily
    missingok
    rotate 30
    compress
    delaycompress
    notifempty
    copytruncate
}
```

## 备份策略

### 数据库备份

```bash
#!/bin/bash
# /opt/univona/backup.sh

BACKUP_DIR="/var/backups/univona"
DATE=$(date +%Y%m%d_%H%M%S)
RETAIN_DAYS=30

mkdir -p "$BACKUP_DIR"

# 全量备份
pg_dump -U univona -Fc univona > "$BACKUP_DIR/univona_${DATE}.dump"

# 压缩
gzip "$BACKUP_DIR/univona_${DATE}.dump"

# 清理旧备份
find "$BACKUP_DIR" -name "*.dump.gz" -mtime +$RETAIN_DAYS -delete

echo "备份完成: univona_${DATE}.dump.gz"
```

### 定时任务

```bash
# 每天凌晨 2 点执行备份
sudo crontab -e -u univona

# 添加以下行
0 2 * * * /opt/univona/backup.sh >> /var/log/univona/backup.log 2>&1
```

### 媒体文件备份

```bash
# 使用 rsync 同步媒体文件到备份存储
rsync -avz --delete /var/lib/univona/media/ /var/backups/univona/media/
```

### 恢复

```bash
# 恢复数据库
gunzip univona_20260215_020000.dump.gz
pg_restore -U univona -d univona -c univona_20260215_020000.dump
```

## 监控

### Prometheus 集成

Admin Server 内置了 Prometheus 兼容的指标收集器（`MetricsCollector`）。在监控系统中配置抓取端点。

Prometheus 配置示例（`prometheus.yml`）：

```yaml
scrape_configs:
  - job_name: 'univona-admin-server'
    scrape_interval: 15s
    static_configs:
      - targets: ['localhost:3001']
    metrics_path: '/api/v1/admin/metrics'
    authorization:
      credentials: 'your-admin-jwt-token'
```

### Grafana 仪表盘

推荐监控以下关键指标：

| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| HTTP 请求速率 | 每秒请求数 | > 1000 req/s |
| HTTP 错误率 | 5xx 响应比率 | > 1% |
| 响应延迟 P95 | 95 分位延迟 | > 500ms |
| 数据库连接池 | 活跃连接数 | > 80% 最大连接数 |
| WebSocket 连接数 | 在线连接数 | 根据容量设定 |
| 磁盘使用率 | 媒体存储使用 | > 80% |
| 内存使用 | 进程 RSS | > 80% 可用内存 |

### 健康检查

Admin Server 提供 `/healthz` 端点用于健康检查：

```bash
# 简单的外部健康检查脚本
curl -sf http://localhost:3001/healthz || echo "服务异常！"
```

## 性能调优

### 连接池

根据并发量调整数据库连接池大小：

```toml
[database]
# 经验公式：max_connections = CPU 核数 * 2 + 磁盘数
# 4 核服务器推荐 10-20 连接
max_connections = 20
```

### 系统参数

```bash
# /etc/sysctl.d/99-univona.conf

# 增加文件描述符限制
fs.file-max = 65536

# TCP 优化
net.core.somaxconn = 65535
net.ipv4.tcp_max_syn_backlog = 65535
net.ipv4.tcp_tw_reuse = 1
net.ipv4.ip_local_port_range = 1024 65535

# 启用 TCP Fast Open
net.ipv4.tcp_fastopen = 3
```

应用：

```bash
sudo sysctl --system
```

### ulimit 配置

编辑 `/etc/security/limits.d/univona.conf`：

```
univona soft nofile 65536
univona hard nofile 65536
univona soft nproc 4096
univona hard nproc 4096
```

## 升级流程

```bash
# 1. 构建新版本
cargo build --release

# 2. 停止服务
sudo systemctl stop univona-admin-server

# 3. 备份当前版本
sudo cp /opt/univona/univona-admin-server /opt/univona/univona-admin-server.bak

# 4. 部署新版本
sudo cp target/release/univona-admin-server /opt/univona/

# 5. 启动服务（数据库迁移自动执行）
sudo systemctl start univona-admin-server

# 6. 验证
curl -sf http://localhost:3001/healthz && echo "升级成功" || echo "升级失败，回滚中..."
```

如验证失败，快速回滚：

```bash
sudo systemctl stop univona-admin-server
sudo cp /opt/univona/univona-admin-server.bak /opt/univona/univona-admin-server
sudo systemctl start univona-admin-server
```
