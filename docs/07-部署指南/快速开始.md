# 快速开始

本指南帮助你在本地或服务器上快速运行 Univona Admin Server。

## 前置要求

- Rust 1.92+（推荐使用 [rustup](https://rustup.rs) 安装）
- PostgreSQL 15+
- Node.js 18+（管理面板构建，可选）

## 1. 克隆仓库

```bash
git clone https://github.com/univona/univona-admin-server.git
cd univona-admin-server
```

## 2. 配置数据库

```bash
# 创建数据库
createdb univona

# 设置连接字符串
export DATABASE_URL="postgres://user:pass@localhost/univona"
```

## 3. 配置环境变量

```bash
# 管理面板 JWT Secret
export JWT_SECRET="your-admin-jwt-secret-here"

# 用户认证 JWT Secret（与 daemon 一致）
export UNIVONA_JWT_SECRET="your-user-jwt-secret-here"

# 管理员 API Key（可选，向后兼容）
export ADMIN_API_KEY="your-admin-api-key"

# 媒体存储路径
export MEDIA_STORAGE_DIR="./media"
```

## 4. 创建 Univona.toml 配置文件

在项目根目录创建 `Univona.toml`：

```toml
host = "0.0.0.0"
port = 8080

[database]
url = "postgres://user:pass@localhost/univona"
max_connections = 10

[security]
cors_origins = ["*"]

[log]
level = "info"
```

> 配置文件会依次尝试加载 `univona-daemon.toml` 和 `Univona.toml`，任意一个存在即可。环境变量 `UNIVONA_` 前缀可覆盖所有配置项。

## 5. 构建和运行

```bash
# 构建
cargo build --release

# 运行（自动执行数据库迁移）
./target/release/univona-admin-server
```

如需启用 Redis 缓存支持：

```bash
cargo build --release --features redis-cache
```

## 6. 构建管理面板（可选）

```bash
cd admin-panel
npm install
npm run build
# 构建产物在 dist/ 目录
```

管理面板使用 `rust-embed` 嵌入到二进制文件中，也可以独立部署静态文件。

## 7. 创建默认管理员

首次启动时，系统会自动创建默认管理员账户：

- 用户名：`admin`
- 密码：`univona`

**请立即在管理面板中修改密码。**

## 验证

```bash
# 健康检查
curl http://localhost:8080/healthz

# 管理员登录
curl -X POST http://localhost:8080/api/v1/admin/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "univona"}'
```

成功后你将获得一个 JWT Token，用于后续管理操作。

## 后台服务

Admin Server 启动后会自动运行以下后台任务：

| 任务 | 间隔 | 说明 |
|------|------|------|
| daemon 自注册 | 启动时 | 将自身注册为 daemon 节点 |
| 心跳上报 | 定期 | 向数据库更新节点状态 |
| 健康检查 | 5 分钟 | 检测所有 daemon 节点可用性 |
| 速率限制清理 | 2 分钟 | 清理过期的限流条目 |
| 离线消息清理 | 定期 | 清理过期的离线消息 |
