# 环境变量完整列表

Univona Admin Server 使用两类环境变量：

1. **Figment 配置变量** - 以 `UNIVONA_` 为前缀，映射到 `Univona.toml` 中的配置项
2. **独立环境变量** - 直接在代码中读取的环境变量，用于敏感信息

## 独立环境变量

以下环境变量在应用代码中直接读取，不通过配置文件系统：

| 变量名 | 必须 | 默认值 | 说明 |
|--------|------|--------|------|
| `JWT_SECRET` | 是 | — | Admin 管理面板 JWT 签名密钥。用于管理员登录生成的 Token 签名。生产环境必须设置为强随机字符串（至少 32 字节） |
| `UNIVONA_JWT_SECRET` | 是 | `univona-dev-secret-change-me` | 用户 JWT 签名密钥。用于用户通过 challenge-response 认证后的 Token 签名。未设置时使用开发默认值并输出警告 |
| `ADMIN_API_KEY` | 否 | — | Admin API Key，用于管理端点的向后兼容认证方式。设置后可通过 `X-Admin-Key` 请求头进行认证 |
| `MEDIA_STORAGE_DIR` | 否 | `./media` | 媒体文件（图片、音频、视频等）的本地存储路径 |
| `RUST_LOG` | 否 | — | Rust 标准日志级别过滤器。优先级高于 `[log] level` 配置项。支持精细控制，如 `univona_admin_server=debug,sqlx=warn` |
| `DATABASE_URL` | 否 | — | 备选的数据库连接字符串。主要用于 SQLx 编译时检查。运行时优先使用 `UNIVONA__DATABASE__URL` 或配置文件 |

## Figment 配置变量

以下环境变量通过 Figment 配置框架加载，使用 `UNIVONA__` 前缀和 `__` 分隔符：

### 服务器配置

| 变量名 | 必须 | 默认值 | 说明 |
|--------|------|--------|------|
| `UNIVONA__HOST` | 否 | `0.0.0.0` | 服务监听地址 |
| `UNIVONA__PORT` | 否 | `3001` | 服务监听端口 |

### 数据库配置

| 变量名 | 必须 | 默认值 | 说明 |
|--------|------|--------|------|
| `UNIVONA__DATABASE__URL` | 是 | — | PostgreSQL 连接字符串。格式：`postgres://user:pass@host:port/dbname` |
| `UNIVONA__DATABASE__MAX_CONNECTIONS` | 否 | `10` | 数据库连接池最大连接数 |

### Redis 配置

| 变量名 | 必须 | 默认值 | 说明 |
|--------|------|--------|------|
| `UNIVONA__REDIS__URL` | 否 | — | Redis 连接地址。需启用 `redis-cache` feature 编译 |

### 日志配置

| 变量名 | 必须 | 默认值 | 说明 |
|--------|------|--------|------|
| `UNIVONA__LOG__LEVEL` | 否 | `info` | 日志级别：`trace`、`debug`、`info`、`warn`、`error` |
| `UNIVONA__LOG__JSON` | 否 | `false` | 是否输出 JSON 格式日志 |

### 安全配置

| 变量名 | 必须 | 默认值 | 说明 |
|--------|------|--------|------|
| `UNIVONA__SECURITY__CORS_ORIGINS` | 否 | `["http://localhost:3000"]` | CORS 允许源列表（JSON 数组格式字符串） |

### 推送通知配置

| 变量名 | 必须 | 默认值 | 说明 |
|--------|------|--------|------|
| `UNIVONA__PUSH__FCM__PROJECT_ID` | 否 | — | Google Cloud 项目 ID |
| `UNIVONA__PUSH__FCM__SERVICE_ACCOUNT_PATH` | 否 | — | FCM 服务账号 JSON 文件路径 |
| `UNIVONA__PUSH__APNS__KEY_PATH` | 否 | — | APNs .p8 私钥文件路径 |
| `UNIVONA__PUSH__APNS__KEY_ID` | 否 | — | Apple Developer Key ID |
| `UNIVONA__PUSH__APNS__TEAM_ID` | 否 | — | Apple Developer Team ID |
| `UNIVONA__PUSH__APNS__BUNDLE_ID` | 否 | — | App Bundle ID |
| `UNIVONA__PUSH__APNS__SANDBOX` | 否 | `false` | 是否使用 APNs 沙盒环境 |

## 完整汇总

按功能分组的所有环境变量快速参考：

### 认证密钥

```bash
# 管理面板 JWT（必须，生产环境强制）
export JWT_SECRET="$(openssl rand -base64 32)"

# 用户认证 JWT（必须，生产环境强制）
export UNIVONA_JWT_SECRET="$(openssl rand -base64 32)"

# 管理 API Key（可选，向后兼容）
export ADMIN_API_KEY="your-admin-api-key"
```

### 数据库

```bash
# 数据库连接（必须）
export UNIVONA__DATABASE__URL="postgres://univona:password@localhost:5432/univona"
export UNIVONA__DATABASE__MAX_CONNECTIONS="20"
```

### 服务器

```bash
# 监听配置
export UNIVONA__HOST="127.0.0.1"
export UNIVONA__PORT="3001"
```

### 存储

```bash
# 媒体文件目录
export MEDIA_STORAGE_DIR="/var/lib/univona/media"

# Redis 缓存
export UNIVONA__REDIS__URL="redis://localhost:6379"
```

### 日志

```bash
# 基础日志级别
export UNIVONA__LOG__LEVEL="info"
export UNIVONA__LOG__JSON="true"

# 精细日志控制（覆盖上述配置）
export RUST_LOG="univona_admin_server=debug,tower_http=debug,sqlx=warn"
```

### 安全

```bash
# CORS 源
export UNIVONA__SECURITY__CORS_ORIGINS='["https://your-domain.com"]'
```

### 推送通知

```bash
# FCM（Firebase Cloud Messaging）
export UNIVONA__PUSH__FCM__PROJECT_ID="your-project-id"
export UNIVONA__PUSH__FCM__SERVICE_ACCOUNT_PATH="/etc/univona/fcm.json"

# APNs（Apple Push Notification service）
export UNIVONA__PUSH__APNS__KEY_PATH="/etc/univona/apns.p8"
export UNIVONA__PUSH__APNS__KEY_ID="ABCDE12345"
export UNIVONA__PUSH__APNS__TEAM_ID="TEAM123456"
export UNIVONA__PUSH__APNS__BUNDLE_ID="com.your.app"
export UNIVONA__PUSH__APNS__SANDBOX="false"
```

## 优先级说明

当同一配置项在多处定义时，优先级从高到低为：

1. `UNIVONA__` 前缀的环境变量
2. `Univona.toml` 配置文件
3. `univona-daemon.toml` 配置文件
4. 代码内置默认值

独立环境变量（`JWT_SECRET`、`ADMIN_API_KEY` 等）不参与此优先级链，它们直接在代码中读取。
