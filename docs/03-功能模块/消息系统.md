# 消息系统

## 模块概述

消息系统是 Univona 的核心通信模块。支持加密消息和明文消息的发送与接收，提供消息编辑、删除（软删除）、置顶、已读状态追踪和表情反应等功能。消息通过 WebSocket 实时推送，并支持游标分页的历史消息查询。

## 核心概念

### 消息类型

消息通过 `message_type` 字段区分类型，默认为 `text`。消息内容根据频道加密策略存储为加密内容或明文：

- **加密消息** (`is_encrypted = true`)：`encrypted_content` 存储密文，`plaintext_content` 为 NULL
- **明文消息** (`is_encrypted = false`)：`encrypted_content` 存储原始内容，`plaintext_content` 同时存储明文（支持全文搜索）

### 消息投递

- **在线投递**：通过 WebSocket 实时推送给频道在线成员
- **离线投递**：通过离线中继存储（详见[离线消息中继](./离线消息中继.md)）
- **消息序列号**：每条消息分配频道级别的单调递增序列号 `seq_num`

### 软删除

删除消息时不物理删除数据库记录，而是设置 `deleted_at` 时间戳。查询时自动过滤已删除消息。

### 置顶消息

管理员可以将重要消息置顶。置顶通过 `pinned_messages` 关联表和消息的 `is_pinned` 标记双重记录。

### 已读状态

通过 `read_states` 表追踪每个用户在每个频道的最后已读消息和未读计数。

### 表情反应

用户可以对消息添加 emoji 反应。每个用户对同一消息的同一 emoji 只能反应一次。

## 数据模型

### messages 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键（UUIDv7，时间有序） |
| `channel_id` | UUID | 所属频道 |
| `sender_id` | UUID | 发送者 |
| `message_type` | VARCHAR(32) | 消息类型，默认 `text` |
| `encrypted_content` | BYTEA | 加密/原始内容 |
| `plaintext_content` | TEXT | 明文内容（仅 Mode B） |
| `is_encrypted` | BOOLEAN | 是否端到端加密 |
| `seq_num` | BIGINT | 频道内序列号 |
| `server_timestamp` | TIMESTAMPTZ | 服务器时间戳 |
| `client_timestamp` | BIGINT | 客户端时间戳（毫秒） |
| `reply_to` | UUID | 回复的消息 ID |
| `is_pinned` | BOOLEAN | 是否被置顶 |
| `edited_at` | TIMESTAMPTZ | 编辑时间 |
| `deleted_at` | TIMESTAMPTZ | 软删除时间 |
| `created_at` | TIMESTAMPTZ | 创建时间 |

### reactions 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键 |
| `message_id` | UUID | 关联消息 |
| `member_id` | UUID | 反应者 |
| `emoji` | VARCHAR(32) | Emoji 字符 |
| `created_at` | TIMESTAMPTZ | 创建时间 |

唯一约束：`(message_id, member_id, emoji)`

### pinned_messages 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `channel_id` | UUID | 频道 ID（联合主键） |
| `message_id` | UUID | 消息 ID（联合主键） |
| `pinned_by` | UUID | 置顶操作者 |
| `pinned_at` | TIMESTAMPTZ | 置顶时间 |

### read_states 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `member_id` | UUID | 用户 ID（联合主键） |
| `channel_id` | UUID | 频道 ID（联合主键） |
| `last_read_id` | UUID | 最后已读消息 ID |
| `unread_count` | INTEGER | 未读消息计数 |
| `updated_at` | TIMESTAMPTZ | 更新时间 |

## API 端点

### 消息查询

```
GET /api/v1/channels/{id}/messages?before={cursor}&limit=50  -- 游标分页查询
```

参数说明：
- `before`：游标消息 ID，返回此消息之前的记录
- `limit`：每页数量，默认 50，最大 100

### 消息编辑和删除

```
PUT    /api/v1/messages/{id}     -- 编辑消息（仅发送者本人）
DELETE /api/v1/messages/{id}     -- 删除消息（发送者或管理员）
```

### 置顶消息

```
POST   /api/v1/messages/{id}/pin     -- 置顶消息（需管理员权限）
DELETE /api/v1/messages/{id}/pin     -- 取消置顶（需管理员权限）
GET    /api/v1/channels/{id}/pinned  -- 获取频道置顶消息列表
```

## 配置

```toml
[messages]
# 历史消息查询每页最大数量
max_page_size = 100
# 默认每页数量
default_page_size = 50
# 消息内容最大大小
max_content_size = "64KB"
```

## 示例

### 查询频道消息

```json
// GET /api/v1/channels/{id}/messages?limit=20
// 响应：
{
  "success": true,
  "data": [
    {
      "id": "msg-uuid-1",
      "channel_id": "ch-uuid",
      "sender_id": "user-uuid",
      "message_type": "text",
      "encrypted_content": [104, 101, 108, 108, 111],
      "is_encrypted": false,
      "plaintext_content": "hello",
      "server_timestamp": 1700000000000,
      "client_timestamp": 1699999999000,
      "reply_to": null,
      "created_at": 1700000000000
    }
  ]
}
```

### 编辑消息

```json
// PUT /api/v1/messages/{id}
{
  "encrypted_content": [104, 105],
  "plaintext_content": "hi (已编辑)"
}
```

```json

  "message_id": "msg-uuid-1",
  "channel_id": "ch-uuid",
  "timestamp": 1700000500000
}
```

