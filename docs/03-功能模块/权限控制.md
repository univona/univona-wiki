# 权限控制

## 模块概述

Univona 使用基于角色的访问控制（RBAC）系统管理用户权限。角色按层级排列，高级角色自动继承低级角色的所有权限。权限检查同时在全局级别和频道级别生效。

## 核心概念

### 角色层级

角色从低到高排列，高级角色拥有低级角色的所有权限：

```
Guest (0) < Member (1) < Moderator (2) < Admin (3) < Owner (4)
```

| 角色 | 等级 | 说明 |
|------|------|------|
| `guest` | 0 | 访客，最低权限，仅可查看频道 |
| `member` | 1 | 普通成员，可发消息、上传媒体 |
| `moderator` | 2 | 版主，可管理消息和成员 |
| `admin` | 3 | 管理员，可管理频道、角色和加密策略 |
| `owner` | 4 | 所有者，拥有最高权限，可管理服务器 |

### 权限列表

#### 消息权限

| 权限 | 最低角色 | 说明 |
|------|----------|------|
| `SendMessage` | Member | 发送消息 |
| `DeleteOwnMessage` | Member | 删除自己的消息 |
| `DeleteAnyMessage` | Moderator | 删除任何人的消息 |
| `PinMessage` | Moderator | 置顶消息 |

#### 频道权限

| 权限 | 最低角色 | 说明 |
|------|----------|------|
| `ViewChannel` | Guest | 查看频道（所有人） |
| `CreateChannel` | Admin | 创建频道 |
| `UpdateChannel` | Admin | 更新频道信息 |
| `DeleteChannel` | Admin | 删除频道 |

#### 成员权限

| 权限 | 最低角色 | 说明 |
|------|----------|------|
| `InviteMember` | Admin | 邀请新成员 |
| `KickMember` | Moderator | 踢出成员 |
| `BanMember` | Moderator | 封禁成员 |
| `ManageRoles` | Admin | 管理角色分配 |

#### 媒体权限

| 权限 | 最低角色 | 说明 |
|------|----------|------|
| `UploadMedia` | Member | 上传媒体文件 |
| `ManageMedia` | Admin | 管理所有媒体文件 |

#### 管理权限

| 权限 | 最低角色 | 说明 |
|------|----------|------|
| `ViewAuditLog` | Admin | 查看审计日志 |
| `ManageServer` | Owner | 管理服务器配置 |
| `ManageChannelEncryption` | Admin | 管理频道加密策略 |
| `ViewContentMonitor` | Admin | 查看内容监控 |

### 权限覆盖机制

除了基于角色的默认权限外，Univona 支持通过 `permission_overrides` 表对单个角色的特定权限进行覆盖。

**工作原理：**

1. 权限检查时，先查询 `permission_overrides` 表是否有对应的覆盖记录
2. 如果存在覆盖，使用覆盖值（`granted = true/false`）
3. 如果不存在覆盖，回退到角色的默认权限（`Role::has_permission()`）

**安全兜底：**

- Owner 角色的权限始终硬编码为全部授予，不可通过覆盖表修改
- 权限管理 API 的权限检查使用硬编码方式（`require_manage_server_hardcoded`），不经过覆盖表，防止管理员意外锁死自己

**向后兼容：**

- `permission_overrides` 表为空时，所有权限完全按默认角色层级生效
- 无需任何配置变更即可启用权限覆盖功能

### 频道级别权限

除全局角色外，用户在每个频道中可以拥有独立的角色。频道成员的角色通过 `channel_members.role` 字段存储，与全局 `members.role` 独立。

### 角色变更规则

- 只能由同级或更高级角色的用户执行角色变更
- 不能将用户提升到高于自身的角色
- `Owner` 角色只能由当前 Owner 转让
- 角色变更记录到审计日志

## 数据模型

### 全局角色

存储在 `members` 表的 `role` 字段中：

```sql
-- members 表
role VARCHAR(32) NOT NULL DEFAULT 'member'
-- 可选值：guest, member, moderator, admin, owner
```

### 频道角色

存储在 `channel_members` 表的 `role` 字段中：

```sql
-- channel_members 表
role VARCHAR(32) NOT NULL DEFAULT 'member'
```

### 封禁记录

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键 |
| `member_id` | UUID | 被封禁用户 |
| `channel_id` | UUID | 频道 ID（NULL 表示全局封禁） |
| `banned_by` | UUID | 操作者 |
| `reason` | TEXT | 封禁原因 |
| `expires_at` | TIMESTAMPTZ | 过期时间（NULL 表示永久） |
| `created_at` | TIMESTAMPTZ | 创建时间 |

### 审计日志

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键 |
| `actor_id` | UUID | 操作者 |
| `action` | VARCHAR(64) | 操作类型 |
| `target_type` | VARCHAR(32) | 目标类型 |
| `target_id` | UUID | 目标 ID |
| `details` | JSONB | 详细信息 |
| `created_at` | TIMESTAMPTZ | 操作时间 |

## API 端点

### 角色管理

```
PUT /api/v1/channels/{id}/members/{member_id}/role  -- 变更频道成员角色
```

### 封禁管理

```
POST   /api/v1/admin/bans               -- 创建封禁
GET    /api/v1/admin/bans               -- 列出封禁记录
DELETE /api/v1/admin/bans/{ban_id}      -- 解除封禁
```

### 频道管理操作

```
POST   /api/v1/channels/{id}/kick/{member_id}    -- 踢出成员
POST   /api/v1/channels/{id}/mute/{member_id}    -- 禁言成员
DELETE /api/v1/channels/{id}/mute/{member_id}    -- 解除禁言
```

### 审计日志

```
GET /api/v1/admin/audit-log    -- 查询审计日志
GET /api/v1/admin/stats        -- 服务器统计信息
GET /api/v1/admin/members      -- 列出所有成员
```

### 权限管理

```
GET    /api/v1/admin/permissions                    -- 获取完整权限矩阵（管理面板）
PUT    /api/v1/admin/permissions                    -- 批量更新权限覆盖（管理面板）
DELETE /api/v1/admin/permissions/{role}/{permission} -- 删除覆盖恢复默认（管理面板）
GET    /api/v1/community/permissions                -- 获取完整权限矩阵（用户端）
PUT    /api/v1/community/permissions                -- 批量更新权限覆盖（用户端）
DELETE /api/v1/community/permissions/{role}/{permission} -- 删除覆盖恢复默认（用户端）
```

## 配置

```toml
[rbac]
# 新注册用户默认角色
default_role = "member"
# 是否允许自助注册
self_registration = true
```

## 示例

### 权限检查

```rust
use univona_shared::rbac::{Role, Permission};

let role = Role::Moderator;

// 版主可以踢人
assert!(role.has_permission(Permission::KickMember));     // true

// 版主不能创建频道
assert!(!role.has_permission(Permission::CreateChannel));  // false

// 管理员可以管理加密策略
let admin = Role::Admin;
assert!(admin.has_permission(Permission::ManageChannelEncryption)); // true
```

### 变更角色

```json
// PUT /api/v1/channels/{id}/members/{member_id}/role
{
  "role": "moderator"
}
```

### 创建封禁

```json
// POST /api/v1/admin/bans
{
  "member_id": "user-uuid",
  "channel_id": "ch-uuid",
  "reason": "违反社区规则",
  "expires_at": "2026-03-15T00:00:00Z"
}
```
