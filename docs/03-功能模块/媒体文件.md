# 媒体文件

## 模块概述

媒体处理模块负责文件的上传、验证、存储、缩略图生成和下载。支持图片、视频、音频和文档类型。上传接口使用 multipart/form-data 格式，单文件最大 50MB。

## 核心概念

### 上传流程

1. 客户端通过 `multipart/form-data` 发送文件
2. 服务端验证文件大小和 MIME 类型
3. 对文件名进行安全清理（仅保留字母、数字、`.`、`-`、`_`）
4. 计算 SHA-256 哈希值
5. 生成 UUIDv7 格式的 `content_id`
6. 存储文件到本地磁盘
7. 如果是图片类型，自动生成 200x200 缩略图
8. 写入数据库记录

### 媒体验证

验证器 (`MediaValidator`) 负责：

- **文件大小验证**：最小 1 字节，最大 50 MB
- **MIME 类型验证**：仅允许白名单内的类型
- **文件名清理**：替换不安全字符为下划线

### 缩略图生成

仅对图片类型文件自动生成缩略图：

- 最大尺寸：200x200 像素
- 输出格式：JPEG
- 缩略图 ID：`{content_id}_thumb`

### 支持的文件类型

| 类别 | MIME 类型 |
|------|-----------|
| 图片 | `image/jpeg`, `image/png`, `image/gif`, `image/webp` |
| 视频 | `video/mp4`, `video/webm` |
| 音频 | `audio/ogg`, `audio/opus`, `audio/mp4` |
| 文档 | `application/pdf` |
| 通用 | `application/octet-stream` |

### 文件大小限制

- 单文件最大：**50 MB** (`50 * 1024 * 1024` 字节)
- 上传路由使用独立的 `RequestBodyLimitLayer`（50 MB），与其他 API 路由（1 MB）分离

## 数据模型

### media_attachments 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键 |
| `uploader_id` | UUID | 上传者 ID |
| `content_id` | VARCHAR(128) | 内容 ID（UUIDv7，唯一） |
| `mime_type` | VARCHAR(128) | MIME 类型 |
| `file_name` | VARCHAR(256) | 清理后的文件名 |
| `file_size` | BIGINT | 文件大小（字节） |
| `width` | INTEGER | 图片/视频宽度 |
| `height` | INTEGER | 图片/视频高度 |
| `duration_ms` | BIGINT | 音视频时长（毫秒） |
| `sha256_hash` | VARCHAR(64) | SHA-256 哈希 |
| `storage_path` | TEXT | 文件存储路径 |
| `thumbnail_path` | TEXT | 缩略图存储路径 |
| `created_at` | TIMESTAMPTZ | 上传时间 |

## API 端点

### 上传

```
POST /api/v1/media/upload    -- 上传文件（multipart/form-data，50MB 限制）
```

multipart 字段：
- `file`（必须）：文件二进制数据
- `file_name`（可选）：覆盖文件名
- `mime_type`（可选）：覆盖 MIME 类型

### 下载

```
GET /api/v1/media/{content_id}              -- 下载原始文件
GET /api/v1/media/{content_id}/thumbnail    -- 下载缩略图
```

### 删除

```
DELETE /api/v1/media/{content_id}           -- 删除文件（仅上传者）
```

## 配置

```toml
[media]
# 文件存储目录
storage_dir = "./data/media"
# 最大文件大小
max_file_size = "50MB"
# 缩略图最大尺寸
thumbnail_max_width = 200
thumbnail_max_height = 200
```

## 示例

### 上传文件

```bash
curl -X POST https://community.example.com/api/v1/media/upload \
  -H "Authorization: Bearer eyJ..." \
  -F "file=@photo.jpg" \
  -F "file_name=my-photo.jpg"
```

```json
// 响应：
{
  "success": true,
  "data": {
    "content_id": "01945a2b-3c4d-7e8f-9a0b-1c2d3e4f5a6b",
    "file_name": "my-photo.jpg",
    "mime_type": "image/jpeg",
    "file_size": 2048576,
    "has_thumbnail": true
  }
}
```

### 下载文件

```bash
# 下载原始文件
curl -O https://community.example.com/api/v1/media/01945a2b.../

# 下载缩略图
curl -O https://community.example.com/api/v1/media/01945a2b.../thumbnail
```

### 删除文件

```bash
curl -X DELETE https://community.example.com/api/v1/media/01945a2b... \
  -H "Authorization: Bearer eyJ..."
```

```json
// 响应：
{
  "success": true,
  "data": null
}
```

删除操作会同时删除原始文件、缩略图（如果存在）和数据库记录。
