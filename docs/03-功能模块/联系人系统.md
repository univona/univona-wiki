# 联系人系统

## 概述

联系人系统提供用户查找、好友请求、接受/拒绝与私聊频道绑定能力。服务端在接受请求后可自动创建私聊频道，客户端使用该频道进行私聊消息发送。

## 核心流程

1. 查找用户（按用户名/ID）
2. 发送好友请求
3. 对方接受后创建私聊频道
4. 双方进入私聊会话

## API 端点

```
POST /api/v1/contacts/lookup
POST /api/v1/contacts/requests
GET  /api/v1/contacts/requests
POST /api/v1/contacts/requests/{id}/accept
POST /api/v1/contacts/requests/{id}/reject
GET  /api/v1/contacts
POST /api/v1/contacts/presence
```

## 数据模型

- `contact_requests`：好友请求记录（from/to/status/channel_id）
- Redis 在线状态缓存：`online:{member_id}`（TTL 300 秒，WebSocket 心跳刷新）

## WebSocket 通知

联系人请求与接受会通过 `SYSTEM` 类型消息通知对方（`message_type=30`），载荷示例：

```json
{
  "type": "contact_request",
  "request_id": "uuid",
  "from_member_id": "uuid",
  "from_username": "alice",
  "from_display_name": "Alice",
  "community_name": "Univona"
}
```

```json
{
  "type": "contact_accepted",
  "channel_id": "uuid",
  "contact_member_id": "uuid",
  "contact_username": "Bob"
}
```

> `from_username`、`from_display_name`、`community_name` 可能为空或缺失（取决于服务端实现）。
